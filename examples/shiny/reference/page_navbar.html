page_navbar

page_navbar(args: NavSetArg, title: Optional[Union[str, Tag, TagList]] = None, id: Optional[str] = None, selected: Optional[str] = None, position: Literal[static-top, fixed-top, fixed-bottom] = 'static-top', header: Optional[TagChildArg] = None, footer: Optional[TagChildArg] = None, bg: Optional[str] = None, inverse: bool = False, collapsible: bool = True, fluid: bool = True, window_title: Union[str, MISSING_TYPE] = MISSING, lang: Optional[str] = None)

Create a navbar with a navs bar and a title.

Parameters

Name Type Description Default
args shiny.types.NavSetArg UI elements. ()
title Optional[Union[str, Tag, TagList]] The browser window title (defaults to the host URL of the page). Can also be set as a side effect via :func:~shiny.ui.panel_title. None
id Optional[str] If provided, will create an input value that holds the currently selected nav item. None
selected Optional[str] Choose a particular nav item to select by default value (should match it’s value). None
position Literal[static-top, fixed-top, fixed-bottom] Determines whether the navbar should be displayed at the top of the page with normal scrolling behavior (“static-top”), pinned at the top (“fixed-top”), or pinned at the bottom (“fixed-bottom”). Note that using “fixed-top” or “fixed-bottom” will cause the navbar to overlay your body content, unless you add padding (e.g., tags.style("body {padding-top: 70px;}")). 'static-top'
header Optional[TagChildArg] UI to display above the selected content. None
footer Optional[TagChildArg] UI to display below the selected content. None
bg Optional[str] Background color of the navbar (a CSS color). None
inverse bool Either True for a light text color or False for a dark text color. False
collapsible bool True to automatically collapse the navigation elements into a menu when the width of the browser is less than 940 pixels (useful for viewing on smaller touchscreen device) True
fluid bool True to use fluid layout; False to use fixed layout. True
window_title Union[str, MISSING_TYPE] The browser’s window title (defaults to the host URL of the page). Can also be set as a side effect via :func:~shiny.ui.panel_title. MISSING
lang Optional[str] ISO 639-1 language code for the HTML page, such as "en" or "ko". This will be used as the lang in the <html> tag, as in <html lang="en">. The default, None, results in an empty string. None
